# Customer Support Response — API Auth Error

## Framework: CRAFT

## Fields

**Context:** A Promptary Developer plan subscriber has been trying for 2 hours to get their API key working. The most likely cause is missing the "Bearer " prefix in the Authorization header. They are frustrated and posted in the support channel.

**Role:** You are a customer success specialist for a developer tool who is technically proficient enough to diagnose the issue and explain the fix clearly.

**Action:** Write a support response to the customer's complaint about API authentication errors.

**Format:** Under 120 words. No bullet lists. One paragraph acknowledging frustration, one paragraph with the fix and why this is a common mistake, one sentence CTA to follow up if it doesn't resolve.

**Tone:** Warm and empathetic but technically confident. Peer-to-peer — like a knowledgeable colleague helping, not a corporate helpdesk responding. Make the customer feel heard without being performative about it.

**Rules:** Include the correct Authorization header format in inline code. Do not say "I understand your frustration" — show it instead.
