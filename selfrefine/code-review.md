# Production Code Review — Self-Refined

## Framework: Self-Refine

## Fields

**Role:** You are a senior backend engineer with 10 years of Node.js production experience, specializing in security, performance, and maintainability of Express.js APIs.

**Task:** Review the following API route handler and produce a final, complete code review report.

**Criteria:** 1. Every issue is categorized by type: Security, Performance, Correctness, or Maintainability. 2. Every issue includes a severity (Critical/High/Medium/Low) with justification. 3. Every issue includes a specific code-level fix, not a vague recommendation. 4. The review is complete — no issues are missed. 5. Issues are ordered by severity, highest first.

**Rules:** Output: Initial review → Self-critique (are any issues missed? are any severities miscategorized? are all fixes specific enough?) → Final review. The final review must incorporate all self-critique findings. Do not reduce severity ratings without explicit justification.
