# API Reference Page — Self-Refined

## Framework: Self-Refine

## Fields

**Role:** You are a senior technical writer who has written API documentation used by thousands of developers, known for clarity, completeness, and zero ambiguity.

**Task:** Write the API reference page for the Promptary GET /api/v1/prompts/:id endpoint.

**Criteria:** 1. Includes a working curl example and a JavaScript fetch example with placeholder values in UPPER_CASE. 2. Documents all possible response codes (200, 401, 404, 429) with example JSON for each. 3. Explains the Authorization header format exactly, including the "Bearer " prefix. 4. Is under 400 words excluding code blocks. 5. Uses second person ("you") throughout.

**Rules:** Always output: Draft → Critique (checking each criterion explicitly) → Final. The final version must address every critique point raised. Do not declare the draft "good enough" — always find at least one improvement.
