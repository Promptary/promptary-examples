# Pricing Page Upgrade Bullets

## Framework: APE

## Fields

**Action:** Write feature bullet points for a Developer plan on a SaaS pricing page.

**Purpose:** Convert free users to paid. The free users already understand the product — they use it daily. These bullets should make the value of upgrading immediately obvious, not explain what the product is. The upgrade blocker is not understanding — it's not feeling the API access is worth €9/month yet.

**Expectation:** 5 bullets, max 12 words each. Lead each bullet with the benefit, not the feature name. Active voice. No bullet should start with "Access to" or "Ability to". Make the reader feel what it's like to have API access, not just know about it.
