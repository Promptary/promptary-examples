# Error Message Plain English Rewrite

## Framework: APE

## Fields

**Action:** Rewrite the following technical error messages in plain English.

**Purpose:** These error messages appear to non-technical end users of a SaaS product when things go wrong. The goal is to reduce support tickets by helping users understand what went wrong and what they can do about it — without needing to understand the underlying technology.

**Expectation:** For each error: one sentence explaining what happened (no technical jargon), one sentence telling the user what to do next. Under 25 words total per error. Friendly but not patronizing.
