# Developer Integration Guide

## Framework: Skeleton-of-Thought

## Fields

**Topic:** A complete developer guide for integrating the Promptary REST API into an existing Node.js application. Audience: backend developers who are comfortable with REST APIs but new to Promptary.

**Skeleton points:** 1. Prerequisites and setup
2. Creating your first API key
3. Making your first GET request
4. Handling the response
5. Error handling and rate limits
6. Updating prompts without redeploying
7. Next steps

**Expand instructions:** Expand each section into 2–3 paragraphs. Every section that involves code must include a working JavaScript (Node.js fetch) code example. Section 5 must include a table of error codes and their meanings.

**Output format:** Markdown. H2 for section headings. Code blocks with language tags. Max 1,500 words excluding code. No introduction or conclusion headers — jump straight into prerequisites.

**Rules:** Do not skip any section from the skeleton. Show the skeleton outline first, then expand. All code examples must be copy-paste functional with placeholder values clearly marked in UPPER_CASE.
