# Systematic API Debugging

## Framework: Plan-and-Solve

## Fields

**Task:** Debug why the Promptary REST API is returning 401 errors for a specific user even though their API key was just created and has never been used. The error occurs on all endpoints.

**Plan instructions:** Before touching any code, write a step-by-step debugging plan. List every possible cause of a 401 error in this system, then order the steps from most likely to least likely. Include how to verify each hypothesis.

**Solve instructions:** Work through each step of the plan in order. For each step, state what you are checking, what the expected result is, and what the finding means for narrowing down the cause.

**Output format:** Plan: numbered list of steps with hypothesis and verification method. Execution: one paragraph per step with finding. Root cause statement. Recommended fix with the specific change to make.

**Rules:** Do not skip steps in the plan. If a step cannot be executed without more information, state exactly what information is needed. Do not propose a fix until the root cause is identified.
