# SaaS Pricing Model Decision

## Framework: Tree-of-Thought

## Fields

**Problem:** We need to choose a pricing model for Promptary. Current options under consideration: (1) usage-based pricing per API call, (2) seat-based pricing per user, (3) flat-rate tier pricing (current model). We have 200 free users, 15 paying Developer plan users, and 2 Team plan customers.

**Number of branches:** Explore 3 branches — one for each pricing model listed.

**Evaluation criteria:** Evaluate each option on: (1) revenue predictability, (2) conversion friction for free-to-paid, (3) scalability with user growth, (4) alignment with how developers actually use the product.

**Selection:** Score each option 1–5 on each criterion. Show a scoring table. Recommend the option with the highest total and explain the two most important reasons. Note the biggest risk of the recommended option.

**Rules:** Each branch must be genuinely different — do not restate the same model with minor variations. Show the scoring table before the recommendation. The recommendation must address the biggest risk directly.
